### Core Concept
Custom Agents become Digital Full-Time Employees (Digital FTEs) -- specialized digital workers that handle entire organizational functions with the reliability of human team members. The critical insight is that AI is an amplifier: it accelerates your velocity in whatever direction you're already heading. Clear specifications produce excellent results fast; vague requirements produce terrible results fast. This makes Spec-Driven Development more critical in the AI era, not less.

### Key Mental Models
- **Tools to Teammates to Digital FTEs**: A three-stage progression -- AI as tool (prompt and use output), AI as teammate (reason, remember, act, improve), AI as Digital FTE (own entire functions like customer support, code review, content moderation with human-level reliability).
- **AI as Amplifier**: AI amplifies both good habits AND bad habits. Clear specs + AI = excellent results fast. Vague ideas + AI = confident-looking but subtly wrong code faster than you can catch errors. Discipline becomes more critical with AI, not optional.
- **Vibe Coding + AI = Amplified Chaos**: Development by feel (prompt, look, adjust, repeat without specs or validation) feels productive because AI generates code fast, but compounds bugs, technical debt, and unmaintainability invisibly.
- **The Virtuous Cycle**: Clear specifications -> precise AI execution -> reliable Custom Agents -> Digital FTEs -> multiplied capacity -> larger problems -> even better specifications (compounds with every AI generation)
- **Path A vs Path B Choice**: Path A (treat AI as faster keyboard, vibe code, watch debt compound) vs Path B (master Agent Factory paradigm, write clear specs, build Digital FTEs, multiply capacity systematically).

### Key Facts
- **Digital FTE definition**: A Custom Agent engineered to own a specific function (customer support, code review, content moderation, data analysis) with reliability expected from a human team member
- **Amplification table**: Clear specifications without AI = good results, slow. Clear specifications with AI = excellent results, fast. No testing without AI = fragile code. No testing with AI = extremely fragile code, multiplied.

- Each chapter concept builds on the others: Five Powers need the AI Stack; Digital FTEs need AIFF Standards to be portable; Spec-Driven Development is how you direct them; the business strategy determines how you monetize them

- The gap between Vibe Coders and Spec-Driven Developers widens with every generation of AI tools (early discipline creates compounding advantage)
- Vibe Coding symptoms: subtle bugs per iteration, nobody knows what system does, changes break unexpectedly, technical debt compounds invisibly, codebase becomes unmaintainable

### Common Mistakes
- Thinking AI makes discipline optional because code appears faster (AI amplifies bad habits just as effectively as good ones -- speed without direction is chaos at scale)
- Treating Digital FTEs as metaphor rather than the logical extension of Custom Agents engineered for specific organizational functions with production reliability
- Choosing Path A (faster keyboard) because it feels immediately productive without recognizing the compounding advantage of Path B (systematic specification and agent building)
- Failing to see how all seven lesson concepts integrate into a single system (each concept in isolation is useful but incomplete; the system effect creates the real advantage)
